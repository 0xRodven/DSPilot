# Competitor Analysis Skill

## When to Use
- Researching competing products
- Building feature comparison matrices
- Analyzing competitor pricing strategies
- Identifying market gaps
- Positioning DSPilot against alternatives

## Analysis Framework

### 1. Competitor Identification

```markdown
## Direct Competitors (Same Solution)
Products that solve the exact same problem for Amazon DSPs.

| Competitor | Website | Pricing | Key Differentiator |
|------------|---------|---------|-------------------|
| [Name] | url | $X/mo | Feature Y |

## Indirect Competitors (Alternative Solutions)
- Spreadsheets (Excel, Google Sheets)
- Amazon's native tools
- Generic fleet management software

## Perceived Competitors
What customers THINK are alternatives (even if they're not).
```

### 5. SWOT Analysis

```markdown
## DSPilot SWOT

### Strengths (Internal Positive)
- 60-second import (fastest in market)
- Purpose-built for Amazon DSPs
- Real-time architecture (Convex)
- Coaching pipeline (unique feature)
- WhatsApp integration
- Simple, intuitive UI

### Weaknesses (Internal Negative)
- Newer in market (less brand recognition)
- No mobile app (web only)
- Limited integrations
- Single-language support

### Opportunities (External Positive)
- Growing Amazon DSP market
- Competitors slow to innovate
- Demand for automation
- API integrations (future)
- International expansion

### Threats (External Negative)
- Amazon could build native tools
- Price competition
- Competitor feature copies
- Market saturation
```

```markdown

1. Competitor websites (pricing, features)

3. LinkedIn (company size, hiring)
4. Job postings (tech stack hints)
5. Social media (product updates)
6. App stores (if mobile)
7. Industry forums (user discussions)
```

## DO NOT
- Make claims without evidence
- Ignore smaller competitors
- Assume competitors are static
- Focus only on features (consider UX, support, brand)
- Forget to update analysis regularly
