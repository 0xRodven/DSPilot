# Product Positioning Skill

## When to Use
- Defining or refining product positioning
- Creating messaging hierarchy
- Identifying target segments
- Differentiating from competitors
- Preparing pitch decks or sales materials

## April Dunford's Positioning Framework

### The 5 Components

```markdown
1. COMPETITIVE ALTERNATIVES
   What would customers use if your product didn't exist?

2. UNIQUE ATTRIBUTES
   What features/capabilities do you have that alternatives lack?

3. VALUE
   What value do those capabilities enable for customers?

4. TARGET CUSTOMER CHARACTERISTICS
   Who cares most about that value?

5. MARKET CATEGORY
   What context makes the value obvious?
```

### 4. Target Customer Characteristics

```markdown
## Who Cares MOST About This Value?

### Ideal Customer Profile (ICP)

**Primary: Station Managers**
- Running 25-75 drivers
- Spending 3+ hours/week on tracking
- Frustrated with spreadsheets
- Under pressure to improve DWC
- Tech-comfortable (uses smartphone apps)

**Secondary: DSP Owners**
- Multiple stations
- Need oversight without micromanaging
- Care about Amazon scorecard
- Budget for tools that save time

### Anti-Personas (NOT Good Fits)
- Stations with <10 drivers (spreadsheet is fine)
- Managers who don't coach (tool won't help)
- Organizations requiring enterprise contracts
```

## Testing Positioning

### Validation Questions
```markdown
1. Does this resonate with target customers?
   → "Does this describe your situation?"

2. Is the differentiation believable?
   → "Have you seen anything else like this?"

3. Is the value clear?
   → "What would this be worth to you?"

4. Is the category right?
   → "What would you call this type of product?"
```

### A/B Test Framework
```markdown
## Headline Test

**Version A** (Feature-led):
"Import Amazon DWC Reports in 60 Seconds"

**Version B** (Benefit-led):
"Improve Your DWC Scores by 15%"

**Version C** (Problem-led):
"Stop Tracking Drivers in Spreadsheets"

**Metrics**: Click-through rate, time on page, sign-ups
```

## DO NOT
- Use jargon customers don't understand
- Claim to be "the best" (subjective)
- Position against too many alternatives
- Forget the "so what?" for every feature
- Assume positioning is static (revisit quarterly)
